开发者

我们是谁？

PostgreSQL 数据库开发者是直接参与项目开发的人，而不是使用 PostgreSQL 构建应用或网站的人。我们并不雇佣程序员，而是通过互联网汇聚全球最优秀的数据库开发者，共同推进 PostgreSQL 项目。您可以了解 PostgreSQL 背后的人，并查看 2006 年 PostgreSQL 周年峰会的合影。

这里有什么？

本栏目主要面向 PostgreSQL 开发者。在这里，您可以找到路线图及当前的 TODO 列表、编码和测试信息、CommitFests、邮件列表以及开发者 FAQ。请注意，这里提供的信息不一定已经进入 PostgreSQL 的正式发行版，甚至可能尚未进入开发版本。

如何参与

以下是一些帮助您参与 PostgreSQL 开发的资源：

  • 阅读开发者 FAQ。
  • 浏览 pgsql-hackers 归档 中的相关话题，也可以选择订阅该邮件列表。
  • 加入 PostgreSQL Hacking Discord。

什么是 CommitFest？

CommitFest 是 PostgreSQL 特性开发流程中的一个阶段。在此期间，提交者和贡献者会集中进行审查、测试以及补丁提交。补丁会登记在 CommitFest 应用 中，供 PostgreSQL 开发者跟踪状态。

在一个大版本发布周期中，通常会安排 5 个 CommitFest：七月、九月、十一月、一月和三月。三月的 CommitFest 是该发布周期中的最后一个，随后项目将进入“特性冻结”阶段，并开始 Beta 测试期。“CommitFest 管理者”负责维护 CommitFest 流程，并承担补丁列表的管理职责。

任何人都可以参与 CommitFest，这也是开始为 PostgreSQL 做贡献的极佳方式。您可以提交补丁并纳入 CommitFest，报名审查补丁，或者成为 CommitFest 管理者（不过该角色通常需要一定的社区经验）。

Google 编程之夏

PostgreSQL 项目长期参与 Google 编程之夏计划。如果您有兴趣参与 PostgreSQL 相关项目，请查看我们的 Summer of Code 页面。
