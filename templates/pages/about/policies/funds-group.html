PostgreSQL 资金小组

最后更新：2019年3月1日。{%git_changes_link%}

章程

PostgreSQL 项目与 Software in the Public Interest, Inc. 之间关系的治理章程。

核心团队成员

PostgreSQL 项目的核心团队由贡献者页面上当前列出的人员组成。

核心团队自行任命其成员。

筹资小组成员

PostgreSQL 项目的筹资小组（即资金小组）最初由 The PostgreSQL Foundation Inc. 终止时仍活跃的成员组成。此后，筹资小组按照下文所述的提案和投票正常程序自行任命其成员。当前成员名单见此处。

筹资小组选举职位

筹资小组选举一名 SPI 联络人和一名副 SPI 联络人。任命须经核心团队批准（核心团队有否决权）。任命没有明确的期限，但任何一方均可随时通过下文所述的提案和投票正常程序进行更换。

SPI 联络人的职责主要是向 SPI 代表筹资小组的利益和决定。此外，SPI 联络人将持续向 SPI 提交有资格获得 SPI 自动贡献成员资格的人员名单，以表彰他们对 PostgreSQL 项目的贡献。

副 SPI 联络人的职责是在 SPI 联络人长时间不可用的情况下，或应 SPI 联络人或筹资小组的明确请求时，履行 SPI 联络人的职责。

提案和投票

提案必须通过电子邮件在资金小组邮件列表上明确提出，并在讨论和投票前获得附议。

决定需要根据筹资小组投票方式进行讨论和投票方可生效。

投票

方式

投票可以使用以下两种方式之一进行：

  • 在邮件列表上公开投票
    • 任何能清楚表达投票意向的方式均可接受，例如：
      • 赞成：yes/+1/aye/yea/affirmative
      • 反对：no/-1/nay/negative
      • 弃权：abstain/0/pass
  • 通过私人电子邮件向2名指定成员（投票收集员）保密投票

公开投票细则

如果确定采用公开投票方式，所有投票将直接发送到邮件列表。投票期结束后，提出动议的成员应汇总投票结果。任何成员都可以审核公开的投票以验证结果。

保密投票细则

如果确定采用保密投票方式，提出动议的成员可以提议投票收集员。投票收集员必须同意承担此职责。如果他们拒绝请求，必须另行选择成员，直到找到2名同意的成员。如果未特别选择成员，默认投票收集员为 SPI 联络人和副 SPI 联络人。

  • 成员将使用订阅资金小组邮件列表的电子邮件地址将选票发送给两位投票收集员
  • 两位投票收集员将独立统计选票
  • 未同时出现在两份名单上的选票将在联系相关成员允许其重新投票后作废。虽非必须，但电子邮件投票应尽可能使用 PGP 对选票进行签名。
  • 选票是保密的。除投票收集员出于验证目的外，不得分享。
  • 投票收集员应在投票结束前不迟于24小时向资金小组邮件列表发送电子邮件，列出已投票的人员，作为对所有人投票的提醒以及通知选票可能未被收到的成员。
  • 投票结果应在投票结束后24小时内发布到邮件列表。

流程

资金请求

  • 资金请求必须用于报销通常符合资金请求指南的支出。
  • 金额不超过1000美元的资金请求可以通过以下特殊流程处理：
    • 任何成员可以将请求转发到资金小组邮件列表，或者请求可能通过审核直接发送到列表
    • 如果没有异议，联络人可以在72小时后批准并拨付请求
    • 如果有任何异议，请求必须作为其他动议按照下文讨论的方式处理
  • 金额超过1000美元的资金请求必须作为其他动议按照下文讨论的方式处理

动议

  • 资金小组的所有现任成员均可提出动议
  • 任何其他现任成员必须对所有动议进行附议
  • 资金小组的所有现任成员均有投票资格
  • 辩论：
    • 动议获得附议后开始
    • 在邮件列表上进行
    • 自动议获得附议后不得少于72小时结束，除非动议中特别要求了更长的辩论期（任何情况下辩论不得少于72小时）
    • 可以通过一项无辩论动议进行延长，该动议须经附议并通过公开投票方式批准
  • 投票：
    • 在邮件列表上或按下文讨论的方式私下进行
    • 法定人数为5名投票成员，或 (TRUNCATE(TRUNCATE(N/3)/2)*2) + 1（其中 N 为活跃成员数），取较大值——例如，有18名活跃成员时，(TRUNCATE(TRUNCATE(N/3)/2)*2) + 1 = 7，因此法定人数为7
    • 由投票成员的简单多数决定
    • 允许选择“弃权”——“弃权”票在法定人数方面计为参与
    • 自辩论结束后保持开放72小时，除非动议中特别要求了更长的投票期（任何情况下投票不得少于72小时）
  • 方式
    • 投票方式可由提出动议的成员决定
    • 如果未指定投票方式，则在邮件列表上采用公开投票方式进行
    • 每次投票只使用一种方式。

资金请求

请求资金

资金请求类型

资金小组资金请求示例：

  • 参加技术会议或展会的差旅
    • 我们已为各种 PostgreSQL 相关会议的演讲者提供差旅费用资助
  • 周边物品（T 恤、文件夹、传单等）
  • 运营成本
    • PostgreSQL 社区基础设施（用于 web、电子邮件、git 等）
  • 专业服务（视频编辑、平面设计）

如何请求资金

要请求资金，请发送电子邮件至 funds-group@lists.postgresql.org ，描述：

  • 您为何请求资金
  • 请求资金的预估总额

资金批准的考量因素

如果请求资金用于提供演讲：

  • 演讲在哪里举行
  • 演讲的预期参加人数
  • 费用
  • 您是否会在 FOSS 或 Creative Commons 许可证下提供演讲内容

如果请求资金用于活动：

  • 活动是否收费
  • 收入将用于何处
  • 活动的预期成果是什么？
  • 对社区有什么益处？

专业服务资金

如果资金请求用于专业服务（如视频编辑、平面设计等），将根据以下（不一定完整的）标准进行评估：

  • 对社区有什么益处
  • 至少一个竞争性报价
    • 每个供应商的参考资料
    • 您对选择哪个供应商的建议及原因
  • 提供所选供应商的书面报价
    • 报价应包括将提供的确切内容
    • 我们有多大的调整空间（执行编辑、移动、添加、更改）
    • 服务何时交付
    • 交付物是什么

我们不一定寻求最便宜的方案，而是寻求总体最优方案。如果某个供应商拥有明显优越的业绩记录（例如），我们可能愿意支付更多费用。
