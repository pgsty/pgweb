新闻与活动审批政策

一般规则

postgresql.org 首页包含新闻和活动列表，这些列表完全为 PostgreSQL 社区的利益而维护。 由于这些版位若商业化提供会非常有价值（每条公告最高可达 400 美元）， 管理员对首页允许发布的内容拥有完全裁量权。

所有待审批的新闻和活动（包括培训活动）必须包含以下信息：

  • 准确的联系信息
  • 能够将该条目与类似条目区分开来的描述性标题
  • 10至25个词的条目摘要，提供足够的信息让 RSS 读者清楚了解是否需要更多信息。
  • 对新闻或活动的描述，清楚地解释它是什么以及它与 PostgreSQL 的关系。
  • 描述中的链接不得使用网址缩短器或混淆器。
  • 标题中不得使用 Emoji 或其他类型的特殊字符。

其他一般规则包括：

  • 所有条目必须以某种直接且明显的方式与 PostgreSQL 相关。
  • 仅含链接的活动或新闻永远不会被接受。
  • PostgreSQL 项目的网络团队可以永久禁止任何因社区成员对准确性、 道德或合法性提出实质性投诉的组织。
  • 网络团队不承诺及时审批公告。如果在特定日期发布对您至关重要， 您需要在提交之前联系 pgsql-www 邮件列表或核心团队成员。
  • 赞助商页面上列出的公司可以在网络团队的自行决定下， 获得新闻和活动的任何限制（如允许的发布频率）的豁免。
  • 网络团队保留部分或全部重写新闻或活动帖子以提高清晰度的权利。
  • 从 PostgreSQL 发布日（包括任何 Beta 或候选发布版本（RC））的 UTC 午夜开始， 在此后72小时的 UTC 午夜之前，不会批准任何新的公告请求。例如，如果 PostgreSQL 新版本于 2018-02-08 发布，则在 2018-02-08 0:00 UTC 到 2018-02-11 0:00 UTC 期间不会批准其他公告。在此期间提交的公告在获得批准后应更新其时间戳， 以免在发布时看起来像旧消息。

总体而言，postgresql.org 首页完全为非营利性的成员所有制 PostgreSQL 社区的利益而运营。 它不是广告平台，网站管理员有权以任何理由或无理由拒绝他们认为与社区需求不一致的任何通知。

审批新闻和公告

PostgreSQL 核心新闻

所有条目除非不准确，否则都将被批准。以下类型的新闻通常是合适的：

  • 所有发布版本，包括主版本、小版本和 Beta 版本
  • 开发计划里程碑
  • 新的认证、奖项和基准测试
  • 核心团队和提交者变更
  • 安全问题
  • 协助/参与征集
  • PostgreSQL 基础设施、网站或法律结构的重大变更
  • 重大新推广工作

PostgreSQL “大家庭”新闻

这包括关于 PostgreSQL 附加项目和驱动程序、主要基于 PostgreSQL 的开源项目， 以及 PostgreSQL 区域和用户组的新闻。

以下条目将被批准，不限频率：

  • 主版本的任何发布或首次 Beta 版
  • 任何新用户组、区域组或网站的创建
  • 关键安全补丁公告
  • 重要的新文章或文档
  • 不频繁的协助/参与征集

其他类型的新闻，如领导层变更，将不会被批准。

专有和外部开源项目新闻

这包括所有产品包含、支持或基于 PostgreSQL 的公司的新闻， 以及将 PostgreSQL 作为多个数据库选项之一来支持的开源项目。

以下类型的新闻将被批准：

  • 支持或围绕 PostgreSQL 的全新产品/项目/服务
  • 关于 PostgreSQL 或包含大量（20%或以上）PostgreSQL 内容的新出版书籍。
  • 现有产品首次支持 PostgreSQL
  • 主版本发布（每季度不超过一次）
  • 如果认为会影响许多 PostgreSQL 社区成员的关键安全更新

其他类型的公告，包括小版本发布、定价变更、领导层变更、 第三方新闻报道、合作伙伴关系以及并购，将被拒绝。

此外，凡未以资金、实物或代码形式支持 PostgreSQL 项目的商业实体， 每 6 个月内仅可发布一条任意类型的公告。

审批活动和活动相关新闻（不包括培训）

活动必须包含重要的 PostgreSQL 相关内容才能被列出。

PostgreSQL 会议和活动

任何半年或更低频率的非培训活动，如果主要关于 PostgreSQL 或面向 PostgreSQL 用户， 将有权被列出。此外，主要或完全关于 PostgreSQL 的会议有权在以下里程碑发布新闻条目， 前提是每个里程碑之间相隔一周或更长时间：

  • 征稿（Call for Papers）
  • 日程发布，
  • 注册开放，
  • 注册截止，
  • 总结回顾。

常规的每月 PostgreSQL 用户组会议将不会被列出。新 PUG 的首次会议可以被列出， 以及不频繁的、可能引起 PUG 正常区域以外人士兴趣的重大特别活动。

开源和数据库会议与活动

开源会议和数据库会议，虽然不是主要关于 PostgreSQL，但包含重要的 PostgreSQL 内容 （如多个演讲场次和一个展位），可以在活动列表中列出。此外，这些活动的征稿 可以作为新闻被接受。

更常规的活动，如每月的本地聚会或小型会议，将不会被列出。 会议必须包含重要的 PostgreSQL 内容才值得列出， 相对重要性的决定由网络团队作出。

培训活动

任何公司的 PostgreSQL 培训都可以在以下条件下被列出：

  • 每条列表必须包含：
    • 该特定培训的具体地点和日期
    • 在线培训注册链接
    • 一句或多句对该特定培训内容的描述

所有描述性文字应纯粹提供信息，不应包含强烈的营销文案或与培训无关的特别优惠。 供应商的网站应列出 PostgreSQL 培训信息，包括即将举行的培训在线日程表。

如果任何培训公司在任何给定季度提交超过4个培训活动， 网站管理员可以（自行决定）拒绝发布该公司的活动。
