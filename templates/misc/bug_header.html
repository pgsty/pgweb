请确保您在报告 Bug 之前已阅读 Bug 报告指南。特别是，请重新阅读 文档以确认您尝试的操作是可行的。如果文档不清楚，请也报告该问题； 这属于文档 Bug。如果程序的行为与文档描述不符，这也是一个 Bug。

低性能不一定是 Bug。请阅读文档或在 邮件列表中寻求帮助来调优您的应用程序。不符合 SQL 标准也不一定是 Bug，除非明确声明了对特定功能的合规性。

在继续之前，请查看 TODO 列表和 FAQ，确认您的 Bug 是否已被记录。如果您无法理解 TODO 列表中的信息，请报告您的问题，以便我们完善 TODO 列表。

如果您认为发现了安全问题，请发送电子邮件至 security@postgresql.org。所有其他 Bug 将被转发到 pgsql-bugs 邮件列表，并将被公开存档。

请确保在报告 Bug 之前，您正在运行所用主要 版本的最新可用小版本。当前支持的版本列表为 {%for ver in supportedversions%}{{ver}}{%if not forloop.last%}, {%endif%}{%endfor%}。

此 Bug 报告表单仅应用于报告 PostgreSQL 数据库的 Bug 和问题。数据库连接器（如 ODBC 和 JDBC）、图形化管理工具（如 pgAdmin）或其他外部项目的问题不应在此报告；请直接向相关项目报告。对于与 PostgreSQL 密切相关的产品，可能有合适的邮件列表可用。

通过此表单提交 Bug，即表示您同意 Bug 报告的所有内容（包括您列出的个人信息）将被发布到公开的 pgsql-bugs 邮件列表，并存档在公开的列表归档中。
